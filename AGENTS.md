# Goblins OS Agent Instructions

These instructions apply to AI agents, automated coding tools, and generated
patches working in this repository.

## Licensing and identity

- Preserve `LICENSE`, `NOTICE`, `TRADEMARKS.md`, and existing copyright,
  attribution, SPDX, license, and provenance notices.
- Do not remove, rewrite, or obscure Goblins OS attribution.
- Do not rebrand the project, release artifacts, package names, domains,
  wallpapers, icons, installer identity, or desktop identity unless the owner
  explicitly requests that change.
- Do not imply that a fork, derivative, generated build, or redistributed image
  is official Goblins OS unless it comes from the official project.
- Modified distributions must use their own name and branding unless written
  permission to use Goblins OS marks exists.

## Security and release integrity

- Never add client-side secrets or live credentials.
- Do not fake release, SBOM, hardware, screenshot, runtime, or checksum proof.
- Keep user-facing copy product-grade. Avoid private planning wording, debug
  labels, implementation jargon, private business language, or operator notes.
